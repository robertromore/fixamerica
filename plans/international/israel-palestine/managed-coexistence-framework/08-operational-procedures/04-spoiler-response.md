## 1. Spoiler Response Principles

### 1.1 Core Philosophy

MCF does not attempt to **defeat** spoilers directly. It **changes the cost-benefit equation**:

| Traditional Approach | MCF Approach |
|---------------------|--------------|
| Deter spoilers through threats | Make interference visible and documented |
| Retaliate against sabotage | Externalize costs to guarantors |
| Negotiate with spoilers | Deny spoilers negotiating leverage |
| Collapse under sustained pressure | Regress procedurally, preserving capacity |

### 1.2 Guiding Principles

| Principle | Description |
|-----------|-------------|
| **Visibility over retaliation** | Document interference rather than respond in kind; legibility creates costs |
| **Cost externalization** | MCF doesn't punish spoilers; guarantors do, in their own domains |
| **Strategic patience** | Spoilers expend resources; MCF outlasts interference cycles |
| **Institutional resilience** | No single point of failure; sabotage triggers containment, not collapse |
| **Response separation** | Distinguish MCF operational response from guarantor punitive response |

### 1.3 Why This Design Works

Traditional peace processes fail against spoilers because:

- Sabotage derails talks entirely (high reward for interference)
- Attribution is contested and deniable
- No consequences for successful disruption
- Process cannot survive sustained pressure

MCF defeats these dynamics by:

| Traditional Failure | MCF Counter |
|---------------------|-------------|
| High reward for disruption | No single capture point; disruption triggers documentation, not collapse |
| Contested attribution | Pattern attribution over time; cumulative record |
| No consequences | Documented interference triggers guarantor action |
| Cannot survive pressure | Rollback preserves capacity; framework waits out interference |

### 1.4 What MCF Does NOT Do

MCF itself does NOT:

- Designate individuals or organizations as spoilers
- Impose sanctions or punishments
- Negotiate with or legitimize spoiler actors
- Retaliate militarily
- Provide intelligence to security services beyond documented patterns

These functions are reserved to parties and guarantors acting in their own capacity.

## 2. Spoiler Categories

### 2.1 Non-State Extremists

#### 2.1.1 Definition

Non-state extremists are **non-governmental actors who use violence or coercion to undermine MCF operations**. This includes actors from both communities.

#### 2.1.2 Identification Criteria

| Criterion | Indicators |
|-----------|------------|
| **Pattern of violence** | Repeated attacks targeting MCF participants, infrastructure, or institutions |
| **Stated opposition** | Public declarations against MCF or coexistence |
| **Coercive activity** | Intimidation campaigns, threats, economic pressure against participants |
| **Sabotage operations** | Deliberate disruption of PCCs, institutions, or services |

#### 2.1.3 Activity Patterns

| Activity Type | Examples |
|---------------|----------|
| **Direct violence** | Attacks on CMS holders, MCF personnel, infrastructure |
| **Intimidation** | Threats, harassment, social pressure against participants |
| **Economic coercion** | Boycotts, employment termination, business pressure |
| **Information operations** | Propaganda, disinformation, rumor campaigns |
| **Infiltration** | Attempts to penetrate MCF institutions |

#### 2.1.4 Response Framework

| Activity Level | MCF Response | Guarantor Action |
|----------------|--------------|------------------|
| **Isolated incidents** | Documentation; shock-absorber activation; protection measures | Monitoring; diplomatic messaging |
| **Sustained campaign** | Enhanced protection; pattern report publication; anti-intimidation escalation | Targeted pressure; asset monitoring |
| **Existential threat** | Operational contraction; phase regression consideration | Direct intervention; security measures |

### 2.2 State-Level Actors

#### 2.2.1 Definition

State-level spoilers are **governments or government-linked actors that systematically undermine MCF through official or unofficial channels**.

This includes:

- External states (most plausibly Iran, but potentially others)
- Party governments acting against MCF obligations
- State intelligence services operating against MCF

#### 2.2.2 Interference Methods

| Method | Description |
|--------|-------------|
| **Proxy support** | Financing, arming, or directing non-state actors |
| **Cyber operations** | Attacks on MCF systems, infrastructure, communications |
| **Information warfare** | State-sponsored disinformation, media manipulation |
| **Diplomatic sabotage** | Undermining guarantor support, international legitimacy |
| **Economic warfare** | Sanctions, trade pressure, financial system manipulation |
| **Passive non-compliance** | Party governments failing to meet obligations |

#### 2.2.3 Identification Patterns

State-level interference is identified through **pattern attribution** over time:

| Signature | Indicators |
|-----------|------------|
| **Weapon signatures** | Types, sources, supply chains |
| **Funding patterns** | Financial flows, currency, intermediaries |
| **Coordination indicators** | Timing, targeting, messaging alignment |
| **Technical signatures** | Cyber attack methods, malware, infrastructure |
| **Information operation patterns** | Narrative consistency, amplification networks |

#### 2.2.4 Response Framework

| Interference Level | MCF Response | Guarantor Action |
|--------------------|--------------|------------------|
| **Opportunistic** | Documentation; pattern monitoring | Diplomatic messaging; monitoring |
| **Systematic** | Enhanced documentation; pattern report publication | Diplomatic pressure; financial measures |
| **Sustained campaign** | Operational adaptation; resilience measures | Sanctions; security measures |
| **Existential threat** | Phase regression; guarantor direct involvement | Security intervention; coalition response |

### 2.3 Internal Obstruction

#### 2.3.1 Definition

Internal obstruction refers to **actions by party governments, officials, or institutions that undermine MCF without formal withdrawal**.

This is distinct from political opposition (which is legitimate) and from operational sabotage (which may trigger other responses).

#### 2.3.2 Obstruction Types

| Type | Description | Examples |
|------|-------------|----------|
| **Passive non-compliance** | Failure to implement obligations | Not processing permits; delaying coordination |
| **Regulatory obstruction** | Using domestic rules to undermine MCF | Creating barriers; selective enforcement |
| **Selective enforcement** | Applying MCF rules unevenly | Protecting some violators; targeting participants |
| **Resource withholding** | Failing to provide agreed resources | Budget delays; personnel shortages |
| **Informal pressure** | Discouraging participation through unofficial channels | Bureaucratic harassment; social pressure |

#### 2.3.3 Response Framework

| Obstruction Level | MCF Response | Guarantor Action |
|-------------------|--------------|------------------|
| **Occasional** | Documentation; party liaison engagement | Monitoring; informal messaging |
| **Pattern emerging** | Formal notification; public reporting | Diplomatic engagement |
| **Systematic** | Compliance report publication; guarantor escalation | Formal pressure; condition adjustment |
| **Fundamental breach** | Guarantor intervention; withdrawal assessment | Treaty enforcement; coalition pressure |

### 2.4 Institutional Capture

#### 2.4.1 Definition

Institutional capture occurs when **MCF institutions are taken over or controlled by actors serving party or factional interests rather than framework objectives**.

#### 2.4.2 Capture Indicators

| Indicator | Warning Signs |
|-----------|---------------|
| **Personnel capture** | Hiring patterns favoring one side; loyalty screening |
| **Process manipulation** | Procedures systematically producing biased outcomes |
| **Information control** | Selective reporting; data suppression |
| **Decision distortion** | Rulings or decisions consistently favoring one party |
| **External direction** | Evidence of instruction from party governments |

#### 2.4.3 Response Framework

| Capture Severity | MCF Response |
|------------------|--------------|
| **Suspected** | Enhanced monitoring; guarantor audit request |
| **Confirmed, localized** | Personnel action; process reform; guarantor oversight |
| **Confirmed, systemic** | Institutional reorganization; external administration |
| **Institutional failure** | Function transfer to guarantors; phase regression consideration |

## 3. Attribution Architecture

### 3.1 Attribution Principles

| Principle | Application |
|-----------|-------------|
| **Pattern over incident** | Single incidents are documented; patterns are attributed |
| **Cumulative record** | Attribution builds over time; record persists |
| **Factual documentation** | Record what happened, not assign blame |
| **Usable by others** | Documentation designed for guarantor and external use |
| **Public transparency** | Pattern reports are published; denies spoilers ambiguity |

### 3.2 Evidence Collection

#### 3.2.1 Evidence Types

| Type | Sources | Handling |
|------|---------|----------|
| **Physical evidence** | Incident sites; weapons; materials | JSVC evidence teams; chain of custody |
| **Documentary evidence** | Communications; financial records; publications | Secure storage; authentication |
| **Testimonial evidence** | Witness statements; insider information | Confidential collection; corroboration |
| **Technical evidence** | Cyber forensics; signal intelligence; imagery | Technical teams; guarantor support |
| **Open source** | Media; social media; public records | Systematic monitoring; verification |

#### 3.2.2 Collection Standards

| Standard | Requirement |
|----------|-------------|
| **Authenticity** | Evidence verified as genuine; chain of custody maintained |
| **Completeness** | All relevant evidence collected; gaps documented |
| **Neutrality** | Evidence collected from all sides; no selective collection |
| **Security** | Evidence protected from tampering; secure storage |
| **Accessibility** | Evidence accessible to authorized reviewers; formatted for use |

### 3.3 Pattern Documentation

#### 3.3.1 Pattern Analysis

JSVC maintains **pattern analysis** across incidents:

| Pattern Element | Analysis |
|-----------------|----------|
| **Actor patterns** | Recurring actors; organizational signatures; network analysis |
| **Method patterns** | Attack types; weapon signatures; operational methods |
| **Targeting patterns** | Target selection; timing; geographic concentration |
| **Narrative patterns** | Messaging; claims of responsibility; propaganda themes |
| **Support patterns** | Funding flows; supply chains; external connections |

#### 3.3.2 Cumulative Record

The **Cumulative Interference Record** is maintained as a living document:

| Component | Content |
|-----------|---------|
| **Incident database** | All documented incidents with available evidence |
| **Actor profiles** | Known actors with documented activity patterns |
| **Pattern assessments** | Periodic assessments of interference patterns |
| **Trend analysis** | Evolution of interference over time |
| **Attribution statements** | Formal pattern attributions when confidence permits |

### 3.4 Publication Standards

#### 3.4.1 Report Types

| Report Type | Content | Audience | Frequency |
|-------------|---------|----------|-----------|
| **Incident reports** | Individual incident documentation | Internal; parties; guarantors | Per incident |
| **Pattern reports** | Analysis of interference patterns | Guarantors; international community | Quarterly |
| **Attribution statements** | Formal pattern attribution | Public | When confidence permits |
| **Annual interference assessment** | Comprehensive annual review | Public | Annual |

#### 3.4.2 Publication Criteria

| Criterion | Standard |
|-----------|----------|
| **Factual accuracy** | All claims verified; speculation clearly labeled |
| **Evidence basis** | Claims supported by documented evidence |
| **Balanced presentation** | Interference from all sources documented |
| **Source protection** | Confidential sources protected |
| **Usability** | Formatted for external use; actionable by guarantors |

## 4. MCF Internal Response

### 4.1 Immediate Response

When spoiler activity is detected, MCF activates immediate response:

#### 4.1.1 Shock-Absorber Activation

Per [Shock-Absorber Protocols](../01-phase-0-institutions/04-shock-absorber-protocols.md):

| Incident Tier | Immediate Response |
|---------------|-------------------|
| **Tier 1-2** | Enhanced monitoring; documentation |
| **Tier 3** | Response ladder activation; party notification |
| **Tier 4** | Full shock-absorber activation; guarantor notification |
| **Beyond Tier 4*** | Crisis protocols; phase regression assessment |

*Shock-Absorber Protocols define Tiers 1-4. Incidents exceeding Tier 4 parameters (extended duration, framework-threatening scope) trigger crisis protocols per [Humanitarian Crisis Protocols](08-humanitarian-crisis-protocols.md) and phase regression assessment per [Phase Regression](03-phase-regression.md).*

#### 4.1.2 Protection Measures

| Measure | Trigger | Authority |
|---------|---------|-----------|
| **Enhanced security** | Threat to specific targets | JSVC Operations |
| **Participant protection** | Threats to CMS holders | Anti-intimidation protocols |
| **Infrastructure protection** | Threats to PCCs or facilities | PCC Directorate |
| **Personnel evacuation** | Imminent threat to MCF personnel | JSVC Executive Director |

#### 4.1.3 Communication Response

| Audience | Timeline | Content |
|----------|----------|---------|
| **MCF personnel** | Immediate | Safety instructions; situation awareness |
| **Parties** | Within 2 hours | Incident notification; coordination request |
| **Guarantors** | Within 4 hours | Situation report; support request if needed |
| **Public** | Per communication protocols | Factual statement; counter-narrative if needed |

### 4.2 Containment Response

If spoiler activity escalates beyond immediate response capacity:

#### 4.2.1 Scope Limitation

| Action | Purpose | Authority |
|--------|---------|-----------|
| **Activity suspension** | Halt specific activities in affected area | JSVC Operations Chief |
| **Geographic isolation** | Limit operations to unaffected areas | JSVC Executive Director |
| **Function prioritization** | Focus on essential functions; defer others | JSVC Executive Director |
| **Resource concentration** | Concentrate resources on maintainable scope | JSVC |

#### 4.2.2 Rollback as Strategic Denial

Phase regression may be invoked as **strategic denial**:

- Spoilers seek total derailment; regression denies this
- Framework survives by shrinking footprint
- Institutional capacity preserved for recovery
- Regression is temporary contraction, not failure

See [Phase Regression Protocols](03-phase-regression.md) for regression procedures.

#### 4.2.3 Institutional Preservation

Even during contraction, MCF preserves:

| Asset | Preservation Method |
|-------|---------------------|
| **Institutional memory** | Records secured; knowledge documented |
| **Personnel capacity** | Core staff retained; expertise preserved |
| **Legal framework** | Legal provisions remain in force |
| **Guarantor relationships** | Communication maintained; support secured |
| **Recovery pathway** | Clear path to re-expansion defined |

### 4.3 Operational Adaptation

MCF adapts operations to reduce spoiler impact:

#### 4.3.1 Vulnerability Reduction

| Vulnerability | Adaptation |
|---------------|------------|
| **Physical access** | Enhanced security; distributed locations |
| **Communication intercept** | Encrypted communications; secure channels |
| **Personnel targeting** | Low-profile operations; rotation; protection |
| **Infrastructure dependence** | Redundancy; alternative systems; mobile capacity |
| **Funding disruption** | Diversified funding; reserve activation |

#### 4.3.2 Resilience Measures

| Measure | Purpose |
|---------|---------|
| **Distributed operations** | No single point of failure |
| **Redundant systems** | Backup for critical functions |
| **Alternative procedures** | Fallback processes if primary disrupted |
| **Surge capacity** | Ability to scale up response when needed |
| **External backup** | Guarantor capacity to assume functions |

## 5. Guarantor Response

### 5.1 Response Separation Principle

**Critical design feature:** MCF does not punish spoilers. Guarantors do, in their own domains.

| MCF Function | Guarantor Function |
|--------------|-------------------|
| Document interference | Act on documentation |
| Publish patterns | Apply consequences |
| Request support | Provide support |
| Maintain operations | Impose costs on spoilers |

This separation:

- Prevents MCF from becoming a battlefield between states
- Ensures interference is not consequence-free
- Preserves MCF neutrality
- Leverages guarantor capacity MCF lacks

### 5.2 Referral Process

When spoiler activity warrants guarantor action:

```text
GUARANTOR REFERRAL PROCESS

Step 1: Documentation
├── JSVC compiles interference record
├── Pattern analysis completed
└── Attribution statement prepared (if applicable)

Step 2: Formal Referral
├── JSVC Executive Director submits referral to guarantor committee
├── Referral includes documentation, pattern analysis, requested action
└── Copy to all guarantors simultaneously

Step 3: Guarantor Review
├── Guarantor committee reviews referral
├── May request additional information
├── Decides on response in their domains

Step 4: Response Coordination
├── Guarantors coordinate response where appropriate
├── MCF informed of intended actions
└── MCF adapts operations as needed

Step 5: Outcome Reporting
├── Guarantors report actions taken
├── MCF documents outcomes
└── Effectiveness assessed
```

### 5.3 Response Categories

#### 5.3.1 Diplomatic Measures

| Measure | Description | Trigger |
|---------|-------------|---------|
| **Private messaging** | Direct communication to spoiler state | Pattern emerging |
| **Demarche** | Formal diplomatic protest | Sustained pattern |
| **Public statement** | Joint guarantor statement | Significant interference |
| **International referral** | UN, regional body engagement | Major interference |
| **Coalition building** | Broadening pressure coalition | Continued interference |

#### 5.3.2 Financial Measures

| Measure | Description | Trigger |
|---------|-------------|---------|
| **Asset monitoring** | Track spoiler-linked financial flows | Pattern emerging |
| **Targeted sanctions** | Sanctions on individuals/entities | Documented role |
| **Broad sanctions** | Sector or economy-wide sanctions | State-level campaign |
| **Aid conditionality** | Condition assistance on behavior change | Internal obstruction |
| **Financial isolation** | Banking/payment system exclusion | Severe interference |

#### 5.3.3 Security Measures

| Measure | Description | Trigger |
|---------|-------------|---------|
| **Intelligence sharing** | Share threat information with parties | Ongoing |
| **Security assistance** | Support to parties for counter-spoiler action | Documented threats |
| **Direct protection** | Guarantor security for MCF assets | Imminent threat |
| **Interdiction** | Block weapons/materiel flows | Documented supply |
| **Intervention** | Direct security action | Existential threat |

### 5.4 Coordination Mechanisms

| Mechanism | Purpose | Participants |
|-----------|---------|--------------|
| **Guarantor committee** | Regular coordination | All guarantors |
| **Emergency session** | Crisis response | All guarantors + MCF |
| **Bilateral coordination** | Specific action coordination | Relevant guarantors |
| **Information sharing** | Intelligence and assessment sharing | Guarantors with capacity |
| **Coalition coordination** | Broader international response | Guarantors + partners |

## 6. Information Warfare Response

### 6.1 Information Threat Landscape

Spoilers use information operations to:

| Objective | Methods |
|-----------|---------|
| **Undermine legitimacy** | Accusations of bias, corruption, capture |
| **Deter participation** | Amplify collaboration accusations; intimidation |
| **Manufacture ambiguity** | Conflicting narratives; false flag claims |
| **Provoke overreaction** | Inflammatory content; emotional manipulation |
| **Erode trust** | Sow doubt about MCF institutions and personnel |

### 6.2 Counter-Disinformation Principles

| Principle | Application |
|-----------|-------------|
| **Rapid factual clarification** | Correct false claims quickly with verified facts |
| **Pre-positioned credibility** | Build trust before crisis; don't start from zero |
| **Transparency** | Openness denies spoilers manufactured ambiguity |
| **Narrative discipline** | Consistent messaging; avoid defensive crouch |
| **Platform diversification** | Multiple channels; not dependent on any one platform |

### 6.3 Response Mechanisms

#### 6.3.1 Monitoring and Detection

| Function | Responsibility |
|----------|----------------|
| **Media monitoring** | Systematic tracking of coverage and narratives |
| **Social media monitoring** | Platform monitoring for disinformation |
| **Trend analysis** | Identification of emerging campaigns |
| **Source tracking** | Attribution of information operations |
| **Early warning** | Alert system for coordinated campaigns |

#### 6.3.2 Rapid Response

| Phase | Action | Timeline |
|-------|--------|----------|
| **Detection** | Identify false or misleading content | Continuous |
| **Assessment** | Evaluate reach, threat level, response need | Within 2 hours |
| **Fact compilation** | Assemble verified facts for response | Within 4 hours |
| **Response formulation** | Draft response appropriate to threat | Within 6 hours |
| **Dissemination** | Distribute response through appropriate channels | Within 8 hours |
| **Follow-up** | Monitor response effectiveness; iterate | Ongoing |

#### 6.3.3 Pre-Positioned Materials

MCF maintains pre-positioned information materials:

| Material Type | Purpose |
|---------------|---------|
| **Fact sheets** | Standard information about MCF institutions and procedures |
| **FAQ responses** | Pre-drafted responses to common false claims |
| **Spokesperson materials** | Talking points for authorized spokespersons |
| **Visual materials** | Infographics, videos for public communication |
| **Media kit** | Materials for journalists covering MCF |

### 6.4 Narrative Resilience

#### 6.4.1 Core Narrative

MCF maintains a **core narrative** that frames understanding:

- MCF is procedural, not political
- Participation is voluntary and reversible
- MCF serves civilian needs, not political agendas
- Both communities benefit from stability
- Spoilers threaten everyone, not just MCF

#### 6.4.2 Counter-Narrative Strategies

| Attack Type | Counter-Narrative |
|-------------|-------------------|
| **"MCF is biased"** | Document balanced treatment; publish fairness data |
| **"Participation is collaboration"** | Emphasize voluntariness, reversibility, practical benefits |
| **"MCF undermines sovereignty"** | Highlight party control; MCF is coordination, not governance |
| **"MCF will fail like others"** | Distinguish design features; document stability achieved |
| **"Spoilers represent the people"** | Contrast spoiler violence with civilian participation |

### 6.5 Disinformation Documentation

All disinformation campaigns are documented for:

- Pattern attribution to sources
- Evidence for guarantor action
- Historical record
- Lessons learned

## 7. Escalation and De-escalation

### 7.1 Escalation Criteria

Spoiler response escalates based on:

| Criterion | Indicators | Escalation Trigger |
|-----------|------------|-------------------|
| **Frequency** | Incident rate | Trend above baseline |
| **Severity** | Casualty/damage level | Tier 3+ incidents |
| **Coordination** | Organized vs. opportunistic | Evidence of coordination |
| **State involvement** | External state role | Documented state link |
| **Effectiveness** | Impact on MCF operations | Operational degradation |

### 7.2 Escalation Ladder

| Level | Threat Characterization | MCF Response | Guarantor Response |
|-------|------------------------|--------------|-------------------|
| **1: Baseline** | Isolated, opportunistic incidents | Standard monitoring and documentation | Monitoring |
| **2: Elevated** | Increased frequency; emerging patterns | Enhanced monitoring; protection escalation | Diplomatic messaging |
| **3: Significant** | Coordinated campaign; operational impact | Containment measures; pattern publication | Targeted measures |
| **4: Severe** | Sustained campaign; major disruption | Operational contraction; guarantor surge | Significant measures |
| **5: Existential** | Framework viability threatened | Phase regression; guarantor direct role | Maximum response |

### 7.3 De-escalation Criteria

Response de-escalates when:

| Criterion | Indicators |
|-----------|------------|
| **Reduced frequency** | Incident rate returns to baseline |
| **Reduced severity** | Incidents below Tier 3 |
| **Disruption of coordination** | Organized activity fragmenting |
| **External pressure effect** | Evidence of behavior change |
| **Operational recovery** | MCF operations normalized |

### 7.4 De-escalation Process

| Phase | Actions | Duration |
|-------|---------|----------|
| **Initial de-escalation** | Reduced protective measures; monitoring continues | 30 days |
| **Sustained de-escalation** | Normal operations resume; documentation continues | 60 days |
| **Full normalization** | Standard operating procedures | After 90 days sustained |

### 7.5 Response Proportionality

| Principle | Application |
|-----------|-------------|
| **Proportionate to threat** | Response matches threat level; no overreaction |
| **Graduated** | Escalation proceeds through levels; no skipping |
| **Reversible** | De-escalation possible at each level |
| **Documented** | Escalation decisions recorded and justified |
| **Coordinated** | MCF and guarantor responses aligned |

## 8. Learning and Adaptation

### 8.1 Post-Incident Analysis

Every significant spoiler incident triggers analysis:

| Analysis Type | Content | Timeline |
|---------------|---------|----------|
| **Immediate assessment** | What happened; immediate lessons | Within 72 hours |
| **Detailed analysis** | Root causes; response effectiveness | Within 30 days |
| **Pattern integration** | Update pattern analysis | Within 60 days |
| **Framework implications** | Structural recommendations | Within 90 days |

### 8.2 Pattern Identification

JSVC maintains ongoing pattern identification:

| Pattern Type | Analysis Focus |
|--------------|----------------|
| **Actor evolution** | How spoiler organizations adapt |
| **Method evolution** | New tactics, techniques, procedures |
| **Targeting shifts** | Changes in target selection |
| **Coordination changes** | Network evolution; new alliances |
| **Information evolution** | New narratives; platform shifts |

### 8.3 Framework Hardening

Learning leads to framework improvement:

| Improvement Type | Examples |
|------------------|----------|
| **Vulnerability reduction** | Address identified weaknesses |
| **Procedure enhancement** | Improve response procedures |
| **Capability building** | Develop new response capabilities |
| **Coordination improvement** | Enhance internal and guarantor coordination |
| **Anticipation development** | Predict and prepare for future threats |

### 8.4 Knowledge Sharing

Spoiler response learning is shared:

| Audience | Content | Method |
|----------|---------|--------|
| **Internal MCF** | Full analysis and recommendations | Internal reports; training |
| **Guarantors** | Pattern analysis; capability needs | Briefings; reports |
| **International community** | General lessons; best practices | Published reports |
| **Academic/research** | Anonymized case studies | Research collaboration |
