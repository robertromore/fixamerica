# Open Internet: Root Causes

## Overview

Threats to the open internet stem from structural market failures, regulatory framework weaknesses, political dynamics, and technological characteristics. Understanding these root causes is essential for designing effective solutions.

## Market Structure Problems

### Natural Monopoly Characteristics

| Factor | Effect |
|--------|--------|
| **High fixed costs** | Network infrastructure expensive |
| **Low marginal costs** | Adding users cheap |
| **Economies of scale** | Larger networks more efficient |
| **Right-of-way requirements** | Limited physical access |
| **Last-mile bottleneck** | Single point of control |

### Barriers to Entry

| Barrier | Description |
|---------|-------------|
| **Capital requirements** | $50K+ per mile for fiber |
| **Franchise agreements** | Local government control |
| **Pole attachments** | Access to utility infrastructure |
| **Incumbent advantages** | Existing customer base |
| **Marketing costs** | Building brand awareness |
| **Interconnection** | Access to networks |

### Competition Failures

| Factor | Effect |
|--------|--------|
| **Duopoly structure** | Cable vs. telco at best |
| **Switching costs** | Equipment, contracts, hassle |
| **Information asymmetry** | Hard to compare services |
| **Bundling** | Locks in customers |
| **Geographic monopolies** | No alternative available |

## Regulatory Framework Weaknesses

### Classification Problems

| Issue | Description |
|-------|-------------|
| **Information service** | Light regulation, limited FCC authority |
| **Telecommunications service** | Heavy regulation, strong authority |
| **Binary choice** | No middle ground |
| **Technology-based** | Outdated distinctions |
| **Judicial interpretation** | Courts limit FCC flexibility |

### Jurisdictional Confusion

| Issue | Description |
|-------|-------------|
| **FCC vs. FTC** | Overlapping, uncertain authority |
| **Federal vs. state** | Preemption battles |
| **Sector regulation** | Telecom vs. media vs. tech |
| **International** | Global services, national rules |

### Regulatory Capture

| Factor | Effect |
|--------|--------|
| **Revolving door** | Industry-regulator movement |
| **Industry expertise** | Reliance on regulated parties |
| **Campaign contributions** | Political influence |
| **Lobbying** | Heavy industry presence |
| **Information asymmetry** | Regulators know less than industry |

## Political Economy

### Industry Power

| Factor | Impact |
|--------|--------|
| **Lobbying spending** | $100M+ annually |
| **Campaign contributions** | Both parties receive |
| **Local influence** | Jobs, investment |
| **Advertising revenue** | Media dependence |
| **Think tank funding** | Research influence |

### Partisan Alignment

| Factor | Description |
|--------|-------------|
| **Republican position** | Generally anti-regulation |
| **Democratic position** | Generally pro-net neutrality |
| **Recent shifts** | Conservative anti-Big Tech sentiment |
| **Policy oscillation** | Rules change with administrations |

### Public Choice Dynamics

| Factor | Effect |
|--------|--------|
| **Concentrated benefits** | ISPs gain from weak rules |
| **Diffuse costs** | Consumer harm spread widely |
| **Complexity** | Public doesn't fully understand |
| **Rational ignorance** | Limited voter attention |

### Network Architecture

| Factor | Implication |

| **Deep packet inspection** | Can see content |

### Platform Architecture

| Factor | Implication |
|--------|-------------|
| **Network effects** | Winner-take-all dynamics |
| **Data advantages** | Incumbents have information |
| **API control** | Platform decides integration |
| **Algorithm opacity** | Hidden decision-making |
| **Scale economies** | Larger platforms more efficient |

### Encryption and Circumvention

| Factor | Implication |
|--------|-------------|
| **HTTPS adoption** | Harder to inspect traffic |
| **VPN use** | Can bypass some restrictions |
| **CDN services** | Shift traffic patterns |

## Economic Incentives

### ISP Incentives

| Incentive | Result |
|-----------|--------|
| **Vertical integration** | Favor affiliated content |
| **Rent extraction** | Charge content providers |
| **Scarcity creation** | Data caps, congestion |
| **Bundling** | Lock in customers |
| **Reduce competition** | Maintain market power |

### Platform Incentives

| Incentive | Result |
|-----------|--------|
| **Engagement maximization** | Algorithmic amplification |
| **Advertiser focus** | User not primary customer |
| **Data collection** | Privacy erosion |
| **Lock-in** | Reduce switching |
| **Network effects** | Grow to dominate |

### Content Provider Incentives

| Incentive | Result |
|-----------|--------|
| **Pay for priority** | Willing if helps |
| **Vertical deals** | Exclusive arrangements |
| **Platform dependence** | Accept terms |

## Speech and Moderation Challenges

### Scale Problems

| Factor | Effect |
|--------|--------|
| **Volume** | Billions of posts daily |
| **Speed** | Viral spread in minutes |
| **Languages** | Global content, limited expertise |
| **Context** | Hard to understand nuance |
| **Automation limits** | AI mistakes |

### Conflicting Values

| Tension | Description |
|---------|-------------|
| **Speech vs. safety** | Protect expression or users |
| **Consistency vs. context** | Rules vs. judgment |
| **Transparency vs. gaming** | Explain rules vs. enable abuse |
| **Free vs. moderated** | Open vs. curated |

### Legal Framework Gaps

| Gap | Description |
|-----|-------------|
| **Section 230 breadth** | Broad immunity |
| **First Amendment limits** | Government can't mandate content rules |
| **Private action** | No constitutional constraint |
| **Contract law** | ToS govern relationship |
