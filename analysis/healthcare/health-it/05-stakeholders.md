# Health Information Technology: Stakeholders

## Primary Stakeholders

### Patients

**Interests**

| Interest | Description |
|----------|-------------|
| Access to records | Ability to see, download, and share own health information |
| Care coordination | Providers can share information for seamless care |
| Privacy protection | Control over who sees sensitive information |
| Error prevention | Complete information prevents mistakes |
| Convenience | Single portal, mobile access, no paperwork |

**Current Experience**

| Metric | Reality |
|--------|---------|
| Multiple portals required | Average patient has 3-5 different portals |
| Data aggregation | Nearly impossible for patients |
| Record transfer | Still requires fax or paper at many facilities |
| Medical errors | Incomplete information contributes to 250K+ deaths annually |
| Digital divide | 21M lack broadband, many lack digital literacy |

**Power Analysis**

| Power Source | Assessment |
|--------------|------------|
| Collective purchasing | Limited (don't choose EHR) |
| Political voice | Diffuse, not organized |
| Market exit | Cannot leave healthcare system |
| Regulatory influence | Limited advocacy capacity |
| Litigation | Class actions rare, settlements small |

### Clinicians (Physicians, Nurses, APPs)

**Interests**

| Interest | Description |
|----------|-------------|
| Efficient workflow | Technology that helps rather than hinders care |
| Reduced documentation burden | Less time on screens, more with patients |
| Clinical decision support | Useful alerts without fatigue |
| Information access | Complete patient information at point of care |
| Professional autonomy | Technology supports judgment, not replaces |

**Current Experience**

| Metric | Reality |
|--------|---------|
| Documentation time | 37% of physician time on EHR |
| After-hours work | 1.5-2.5 hours/day additional |
| Alert fatigue | 90%+ of alerts overridden |
| Information access | Often incomplete across providers |
| Burnout rate | 50%+ cite EHR as major factor |

**Power Analysis**

| Power Source | Assessment |
|--------------|------------|
| Professional organizations | Moderate advocacy, conflicting interests |
| Workforce scarcity | Some leverage in employment |
| Media influence | Stories of burnout get attention |
| Implementation influence | Can resist or sabotage systems |
| Unionization | Rare, growing in some areas |

## Secondary Stakeholders

### Healthcare Delivery Organizations

**Hospitals and Health Systems**

| Interest | Description |
|----------|-------------|
| Operational efficiency | Reduce administrative costs |
| Revenue optimization | Capture charges, reduce denials |
| Competitive advantage | Network effects favor integrated systems |
| Risk management | Documentation for liability protection |
| Quality performance | Meet reporting requirements |

**Current Experience**

| Challenge | Impact |
|-----------|--------|
| Implementation costs | $100-500M for major systems |
| Vendor lock-in | Cannot switch, limited leverage |
| Interface costs | Millions to connect systems |
| Staff productivity | Transition periods reduce throughput |
| Cybersecurity | Major financial and reputational risk |

**Power Analysis**

| Power Source | Assessment |
|--------------|------------|
| Purchasing power | Moderate; vendors have leverage |
| Political influence | Significant through associations |
| Regulatory voice | Major input on rulemaking |
| Market position | Varies by system size |

**Physician Practices**

| Interest | Description |
|----------|-------------|
| Affordable technology | Limited capital for IT investment |
| Vendor independence | Flexibility to change systems |
| Revenue cycle | Efficient billing and collections |
| Quality reporting | Meet MIPS/APM requirements |
| Work-life balance | Technology that doesn't extend workday |

**Current Experience**

| Challenge | Impact |
|-----------|--------|
| Resource constraints | Cannot afford best systems |
| Compliance burden | MIPS reporting complexity |
| Payer variation | Multiple portals for authorizations |
| Staff capacity | No dedicated IT support |
| Consolidation pressure | Independent practice declining |

### EHR Vendors

**Epic Systems**

| Characteristic | Detail |
|----------------|--------|
| Market position | 38% hospital market, growing |
| Ownership | Employee-owned, private |
| Revenue | ~$4.6 billion (2023) |
| Workforce | ~13,000 employees |
| Strategy | Comprehensive platform, control ecosystem |

**Interests**

- Maintain market dominance
- Control data and interfaces
- Premium pricing
- Expand scope (payer, analytics, AI)
- Resist regulatory intervention

**Oracle Health (Cerner)**

| Characteristic | Detail |
|----------------|--------|
| Market position | 22% hospital market |
| Ownership | Oracle subsidiary (acquired 2022) |
| Revenue | Part of Oracle Health (~$5B segment) |
| Workforce | ~28,000 employees |
| Strategy | Cloud migration, AI integration, VA contract |

**Interests**

- Compete with Epic
- Complete VA Millennium replacement
- Leverage Oracle cloud infrastructure
- Expand government contracts
- Enterprise healthcare platform

**Other Major Vendors**

| Vendor | Market | Interests |
|--------|--------|-----------|
| Meditech | Community hospitals | Defend market, cloud transition |
| Athenahealth | Ambulatory | Growth, cloud platform |
| eClinicalWorks | Small practices | Volume, market share |
| Altera (Allscripts) | Various | Specialty focus, PE returns |
| NextGen | Ambulatory | Specialty markets |

### Payers (Insurance Companies)

**Interests**

| Interest | Description |
|----------|-------------|
| Administrative efficiency | Electronic transactions reduce costs |
| Prior authorization | Automated approval/denial |
| Data access | Claims, clinical data for utilization |
| Quality measurement | Data for HEDIS, Stars |
| Risk adjustment | Complete diagnosis capture |

**Current Experience**

| Area | Status |
|------|--------|
| Prior authorization | Still largely manual, burdensome |
| Data exchange | Da Vinci implementation growing |
| Quality data | Incomplete clinical data access |
| Interoperability | CMS mandates driving change |

**Power Analysis**

| Power Source | Assessment |
|--------------|------------|
| Market concentration | Top 5 control 45%+ of commercial |
| Regulatory influence | Significant lobbying |
| Payment control | Leverage over providers |
| Data assets | Major data holders |

### Federal Government

**Department of Health and Human Services (HHS)**

| Agency | Role | Authority |
|--------|------|-----------|
| ONC | Standards, certification, policy | HITECH, Cures Act |
| CMS | Payment incentives/penalties | Medicare/Medicaid |
| FDA | Medical device software | FDCA |
| OCR | HIPAA enforcement | Privacy/security |
| CDC | Public health informatics | Surveillance |

**Interests**

| Interest | Description |
|----------|-------------|
| Medicare/Medicaid efficiency | Reduce administrative costs |
| Quality improvement | Data for measurement |
| Public health | Disease surveillance, emergency response |
| Research | Data for NIH, AHRQ research |
| National security | Cybersecurity, pandemic preparedness |

**Current Initiatives**

| Initiative | Status |
|------------|--------|
| TEFCA | Implementation beginning |
| USCDI expansion | Annual updates |
| Information blocking enforcement | Minimal activity |
| AI governance | HTI-1 transparency requirements |
| CDC data modernization | Multi-year program |

### State Governments

**Interests**

| Interest | Description |
|----------|-------------|
| Medicaid operations | Efficient HIT systems |
| Public health | State surveillance, registries |
| Health information exchanges | Many operate state HIEs |
| Licensure | Physician credentialing data |
| Emergency response | Pandemic, disaster coordination |

**Variation**

| Type | Examples |
|------|----------|
| Active HIE states | Indiana, New York, Michigan |
| Limited HIE states | Many smaller states |
| Progressive regulation | California, NY |
| Minimal intervention | Many states |

## Affected but Underrepresented Groups

### Vulnerable Patients

**Digital Divide Populations**

| Population | Barriers |
|------------|----------|
| Elderly (75+) | Digital literacy, vision, dexterity |
| Rural residents | Broadband access, telehealth |
| Low-income | Device access, data plans |
| Homeless | No address, no continuity |
| Incarcerated | System access, transitions |
| Limited English | Portal language limitations |

**Specific Needs**

| Need | Current Gap |
|------|-------------|
| Accessible interfaces | Many portals not ADA compliant |
| Language support | Limited translation |
| Low-literacy content | Medical terminology barriers |
| Caregiver access | Complex proxy processes |
| Continuity tools | Homeless patients cannot access records |

### Safety Net Providers

**Federally Qualified Health Centers (FQHCs)**

| Challenge | Impact |
|-----------|--------|
| Resource constraints | Cannot afford premium systems |
| Legacy systems | Often outdated technology |
| Vendor options | Limited choices for budget |
| Integration | Multiple systems not connected |
| Reporting burden | Complex quality requirements |

**Public Hospitals**

| Challenge | Impact |
|-----------|--------|
| Budget constraints | Underfunded for technology |
| Complex populations | Homelessness, behavioral health |
| Legacy systems | Expensive transitions |
| Staff shortages | Limited IT expertise |

### Public Health Agencies

**Interests**

| Interest | Description |
|----------|-------------|
| Timely reporting | Real-time disease surveillance |
| Complete data | All providers reporting |
| Modern infrastructure | Replace legacy systems |
| Workforce | Trained informatics staff |
| Emergency response | Surge capacity, coordination |

**Current Challenges**

| Challenge | Impact |
|-----------|--------|
| Underfunding | Historic underinvestment |
| Legacy systems | 1990s technology still in use |
| Workforce shortage | Low pay, limited training |
| Provider compliance | Inconsistent reporting |
| Fragmentation | 50 states, 3,000+ local agencies |

### Health IT Startups and Innovators

**Interests**

| Interest | Description |
|----------|-------------|
| Market access | Ability to sell to health systems |
| API access | Connect to EHR data |
| Fair pricing | Reasonable interface fees |
| Standards adoption | Level playing field |
| Investment climate | Viable path to scale |

**Barriers**

| Barrier | Effect |
|---------|--------|
| Vendor gatekeeping | EHR vendors control access |
| App store fees | High costs for marketplace listing |
| Integration complexity | Expensive, time-consuming |
| Sales cycle | 18-36 month hospital decisions |
| Incumbent relationships | IT departments favor existing vendors |

## Stakeholder Dynamics

### Power Imbalances

**Who Has Power**

| Stakeholder | Power Sources |
|-------------|---------------|
| Epic | Market share, data control, network effects |
| Oracle/Cerner | VA contract, Oracle resources |
| Large health systems | Purchasing, political voice |
| Major payers | Payment control, data assets |
| CMS | Payment incentives |

**Who Lacks Power**

| Stakeholder | Power Gaps |
|-------------|------------|
| Patients | Diffuse, not organized |
| Clinicians | Burnout, not united |
| Small practices | No leverage |
| Startups | No market access |
| Public health | Underfunded |

### Coalition Possibilities

**Natural Allies**

| Coalition | Shared Interest |
|-----------|-----------------|
| Patients + Clinicians | Reduce burden, improve care |
| Startups + Public interest | Break vendor lock-in |
| Payers + Employers | Reduce administrative costs |
| Public health + CMS | Population health data |
| Academic centers + ONC | Standards development |

**Unlikely Alliances**

| Parties | Why Unlikely |
|---------|--------------|
| Epic + Interoperability advocates | Business model conflict |
| Large systems + Small practices | Competitive interests |
| Vendors + Price transparency | Revenue implications |
